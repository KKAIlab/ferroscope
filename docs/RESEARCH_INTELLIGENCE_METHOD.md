# FerroScope 研究情报学习规范 v0.8

更新日期：2026-07-23

## 0. 目的

本规范把“读到一篇新论文”改造成可审计的研究情报，而不是写成论文摘要或期刊新闻。

最终要回答三个层次的问题：

1. **论文层**：作者提出了什么问题，真正新增了什么知识，证据链是否支持结论？
2. **团队层**：这项工作位于该团队哪条连续研究主线，能力平台和下一步动作是什么？
3. **个人层**：它对脂质生化、氧化脂质组学和 ferroptosis 研究者具体有什么可操作价值？

本规范借鉴用户提供的三类阅读范例：

- “60 秒顶刊阅读”：Story → Advance → New Knowledge → Evidence → Journal fit → Shift → Open questions；
- “单篇逐图精读”：问题 → 假说 → Figure 因果链 → 多尺度证据 → 机制边界；
- “Chai lab 20 篇文章精读”：Clinical problem → Field problem → Scientific question → Hypothesis → Strategy → Figure chain → Findings → Innovation → Impact → Next questions → Lab synthesis。

范例不是事实来源。所有元数据、结果和优先级判断仍需回到论文、数据库和实验室官网核验。

### 0.1 方法范例与领域证据的硬隔离

三类范例只提供提问顺序、证据拆解方式和跨论文综合方法，不能因为“写得像一篇精读”就进入 ferroptosis 事实库。系统把输入分成三种来源类型：

- `method-example`：用户提供的 60 秒、逐图或跨论文阅读范例；只允许更新本规范和测试规则；
- `domain-evidence`：铁死亡相关原始论文、正式更正、数据库记录和实验室官网；只有这一类可以进入团队和论文数据；
- `search-lead`：二次解读、社交媒体、新闻稿或用户笔记；只能生成待核验问题，不能直接生成 Fact。

无关范例中的蛋白、疾病、Figure 结论和团队名称不得写入 `lab-research*.json`、`labs.json` 或实时情报数据。即使范例内容本身可靠，也只能学习“怎么问”，不能迁移“问到了什么”。

## 1. 三尺度阅读系统

三份参考材料只用于学习思维，不把无关学科内容混入 ferroptosis 数据库：

1. **单篇快速压缩**：用 60 秒框架决定是否值得继续；
2. **单篇机制深挖**：按 Figure 和干预时间尺度重建因果链；
3. **团队纵向综合**：跨多篇论文识别持续问题、能力演化和理论边界。

三个维度是同一条情报的不同分辨率，不是团队分类，也不是三个互相替代的模板。37 个团队都必须有 60 秒级问题判断与团队纵向档案；只有达到升级门的高价值论文需要逐 Figure 深挖。不能为了“每个维度都出现”而虚构 Figure 细节，也不能用团队综述替代单篇证据核验。

### 1.1 60 秒初筛

每篇新记录先回答以下七项；任何一项回答不了，都应进入“待精读”，而不是补写推测。

| 时间 | 问题 | 合格输出 |
|---|---|---|
| 0–10s | Story：作者真正要解决什么矛盾？ | 一个可证伪的问题，不是“研究 X 在 Y 中的作用” |
| 10–20s | Advance：相对旧模型新增了什么？ | Before → Now，并说明比较基线 |
| 20–30s | New knowledge：最小新增知识单元是什么？ | 机制、结构、空间、时间或因果关系 |
| 30–40s | Evidence：哪组实验承担关键因果？ | 干预 → 读出 → 救援/反向验证 → 生理模型 |
| 40–50s | Scope：结论在哪些系统成立？ | 物种、细胞/组织、处理、测定条件 |
| 50–55s | Relevance：为什么值得本项目追踪？ | 对脂质、铁、细胞器、方法或转化的具体接口 |
| 55–60s | Decision：下一步做什么？ | 略读 / 精读 / 追踪 / 暂缓，并给理由 |

禁止仅以期刊名、影响因子或标题中出现 `ferroptosis` 作为精读理由。

### 1.2 重大研究精读

精读按以下顺序进行：

1. **Canonical metadata**：标题、期刊、年份、DOI/PMID、文章版本、作者和通讯单位；
2. **Clinical/biological problem**：论文为什么值得做，但不得把未测试的临床愿景写成已解决问题；
3. **Field gap**：已知、未知、争议分别是什么；
4. **Scientific question**：一个主问题，最多三个子问题；
5. **Hypothesis**：区分作者预设假说与看完结果后的回溯性叙述；
6. **Experimental strategy**：筛选、机制、正交验证、体内/患者层级；
7. **Figure evidence chain**：每个结论对应哪一类实验，避免凭摘要重建虚构 Figure 编号；
8. **Claim–evidence matrix**：必要性、充分性、直接测量、救援、外部重复逐项标记；
9. **Boundary and caveat**：替代解释、模型局限、测定条件和利益冲突；
10. **Open questions**：必须由证据缺口导出，而不是泛泛问“能否临床转化”。

### 1.3 单篇机制深挖

从用户提供的逐图精读范例中只抽取以下通用方法：

- 用急性扰动优先判断直接效应，用长期敲除判断系统后果；
- 区分共定位、相互依赖、直接物理接触与稳定复合物；
- 把 `genome-wide association/enrichment` 明确标为关联层，不写成功能验证；
- 让结构、生化、染色质、转录和表型证据回答同一个机制问题；
- 明确机制能解释的比例和剩余现象，禁止把一个复合 motif 或单通路写成全部解释；
- 体外细胞系统中的机制需要与体内组织状态分层呈现。

### 1.4 三层升级门

三种读法都进入系统，但不是把每条动态重复写三遍：

1. **所有候选信号**先做 60 秒初筛；回答不了核心问题的记录不得进入首页高优先级；
2. **高相关或高不确定信号**才升级为单篇机制深挖，逐图核对关键因果、替代解释与适用边界；
3. **至少两篇原创工作与官网方向完成交叉核验**后，才允许写入团队纵向主线；
4. 新论文如果只重复既有机制，则更新动态但不改写团队主线；只有问题、能力或结论边界发生变化时才改写。

## 2. 团队研究主线的抽取

不能把 20 篇论文简单并列。对每个团队需要形成一条“问题演化链”：

```text
初始矛盾
  → 关键能力/模型建立
  → 第一代机制
  → 机制边界或反例
  → 新工具/新尺度
  → 当前可检验前沿
```

每个团队档案必须包括：

- **核心问题**：团队持续在解决的科学矛盾；
- **能力平台**：该组自己能完成什么，而不是合作中偶然出现什么；
- **至少两项代表工作**：研究问题、结论、关键证据、限制与来源；
- **当前前沿**：最近工作相对旧主线发生了什么变化；
- **监控问题**：未来一年看到什么结果会改变我们对该团队的判断；
- **负面/边界证据**：更正、Editor’s Note、预印本反证、体内外不一致；
- **归属说明**：主导、共同主导或方法合作，不把大合作成果全部归给一个 PI。

## 3. Ferroptosis 专用证据门

### Gate A｜死亡类型

至少组合使用两类证据：

- 遗传学：GPX4、FSP1、ACSL4、NCOA4 等必要性/充分性；
- 药理学：结构不同的抑制剂/救援剂，注意脱靶；
- 铁依赖：可利用铁测量或铁操作，而非只看总铁；
- 脂质过氧化：直接氧化脂质/自由基测量优先于单一 C11-BODIPY；
- 形态与时间：只能作为辅助，不能单独定性。

### Gate B｜脂质机制

必须说明测到的是：脂肪酸、总磷脂、分子种、氧化分子种、自由基还是代理荧光。不得把它们互换。

### Gate C｜空间与通量

涉及膜、溶酶体、线粒体或内质网时，需要区分定位、富集、起始位置和因果必要性。静态共定位不能自动证明通量。

### Gate D｜体内与转化

细胞、类器官、小鼠、离体人器官和患者试验是不同证据层级。临床结局不能自动证明 ferroptosis 是作用机制。

## 4. 三类语句必须分开

每条档案中的句子应属于以下一种：

- **Fact**：论文、数据库或官网直接支持的事实；
- **Inference**：基于多条事实做出的项目判断，必须标注“我们判断/提示”；
- **Unknown**：当前证据不足，不能用流畅措辞填空。

“首次”“推翻”“范式转换”“主流观点”属于高风险词。若没有论文原文、同期评论或系统综述支持，降级为“补充”“修正”“提示”。

## 5. 三类范例带来的审计规则

### 5.1 GPX4 fin-loop 60 秒案例

正式记录为 *Cell*，DOI `10.1016/j.cell.2025.11.014`。原范例中出现“为什么 Nature 会收？”和“Why Science?”，属于模板残留，必须自动报错。

“R152H 催化活性基本正常”只能写成“在论文所用体外活性测定中未显著受损”。论文恰好研究膜锚定，因此不能把可溶体系活性外推为膜内 PLOOH 清除效率正常。

推荐结论：

> 该研究把 GPX4 的保护模型从“只有催化活性”修正为“催化能力与膜接近性共同决定细胞保护”。

不推荐结论：

> GPX4 领域发生了范式革命。

### 5.2 Chai lab 20 篇案例

应保留的优点：问题分层、假说、实验策略、Figure 证据链、下一问题，以及跨论文抽象出的 `Phenotype → Cell → Niche → Mechanism`。

需要修正的偏差：

1. 只选择子刊或高影响因子论文会造成幸存者偏差；方法论文、阴性结果和更正同样重要；
2. “首次”“传统认为”“推翻”需要来源；
3. “未见刊”必须标为 preprint/accepted/in press，不能与正式论文同级；
4. Figure 编号、marker、时间点和谱系贡献必须从原文核对；
5. 团队层综合属于二次分析，不能伪装成作者结论；
6. 大型合作需核对通讯作者、单位和贡献，避免错误归属。

### 5.3 第二批审计暴露出的归属问题

“某 PI 出现在作者列表中”不等于“该论文代表该实验室的研究主线”。每篇代表研究必须新增三个字段：

- `labRole`：团队主导、共同主导、方法协作或待核对，并用一句话说明依据；
- `publicationStatus`：version-of-record、accepted、preprint 或 corrected；
- `readingLevel`：evidence-audited 或 figure-audited。

2026 年精胺—铁螯合论文提供了一个边界测试：Raphaël Rodriguez 团队参与循环伏安测量，但项目领导和通讯作者来自其他团队。因此它可以作为全站最新信号追踪，却不能作为 Rodriguez 团队主导的代表论文。相反，其 CD44–铁内吞和溶酶体铁激活论文能形成团队连续问题链。

同一论文对 Tang／Kang 团队也是更细的边界测试。Daolin Tang 是共同通讯并参与概念、设计和解释，但论文贡献声明把项目规划与总体监督主要归于 Jun Li／Libing Song 团队；Rui Kang 只参与数据分析和修稿。因此本系统允许把论文收入 Tang／Kang 档案，但 `labRole` 必须写成“共同概念合作”，不得写成 UTSW 总体主导。

### 5.4 从实验室名单升级为能力地图

“谁在做 ferroptosis”只是入口，不能直接回答“这条消息为什么对我有用”。37 个团队还要按不可互换的能力模块组织：

- 氧化脂质分子种与自由基化学；
- GPX4／FSP1／CoQ 防御生化；
- 铁稳态、铁成像与 ferritinophagy；
- 肿瘤免疫和微环境；
- 神经、肾脏等疾病模型与人样本；
- 化学探针、化学蛋白质组学和药物发现；
- 细胞死亡方式的定义与方法边界。

这一步防止把 Bayır、Zou、Bush 等相邻但能力完全不同的团队压缩成同一种“铁死亡组”。站点排序应同时回答：研究问题是否相关、证据是否可信、该团队能补足哪一种能力。

方法团队还需额外区分“发现归属”和“分析能力”。例如 Fedorova 团队在脂滴 FSP1 论文中承担 epilipidomics 协作，这能证明其氧化中性脂质分析能力，但不能把整篇脂滴机制归为该团队独立发现。

### 5.5 第一批逐图审计新增的四道边界门

第一批 5 篇逐 Figure 样本覆盖 PUFA 醚磷脂、溶酶体、RPE 组织遗传、PRDX6—硒利用和脂滴 FSP1。逐图核对后，新增四条必须在每张 Figure 层面回答的规则：

1. **观察工具是否同时是干预工具**：自由基探针、铁探针和定位探针可能捕获自由基、氧化 Fe2+ 或改变局部浓度。若探针本身能够救援死亡，其定位结果必须写成“在该探针扰动后的系统中观察到”，不能当作无扰动成像；
2. **工程化遗传压力不能偷换成药理可转化性**：GPX4-null＋Fer-1 withdrawal、诱导敲除和预先构建的 knockout 异种移植可以证明生物学依赖，但不等同于已有体内可用 GPX4 抑制剂，也不等同于患者治疗响应；
3. **疾病表型相似不等于疾病病因**：模型出现 geographic atrophy 样病理、神经退行样 signature 或人类疾病 marker，只能写成“重现部分特征”。除非有人样本、时序和干预证据闭环，不得写成“证明该疾病由 ferroptosis 驱动”；
4. **混合死亡程序必须保留**：当 Fer-1 与 Nec-1s 均能救援，或同时出现多条执行通路信号时，结论应写成“ferroptosis 相关损伤与其他程序并存”，不能为保持主题纯度删去反证。

另有一条化学生物学专用门：体外检出共价中间体不等于细胞内该中间体占主导。PRDX6 体系中 GS-Se-SG 是否为真实主要供体、硒如何转交 SEPHS2，必须保留为 Unknown，直到细胞内定量或直接转移实验完成。

### 5.6 论文身份与团队归属必须分层

一篇论文可能同时出现在多个团队档案中，因为不同团队分别承担机制主导、化学合成、脂质组学或动物验证。此时必须区分两个对象：

- **唯一论文**：以规范化 DOI 为身份；`readingLevel`、正式版本状态和逐 Figure 证据链属于论文层，任何团队档案引用它时都必须共享；
- **团队—论文记录**：保存该团队在论文中的 `labRole`、这项工作如何连接团队主线，以及对该团队的能力证明边界。

因此，站点不能把 74 条团队—论文记录写成 74 篇不同论文，也不能让同一 DOI 在一个团队中显示为逐图精读、在另一个团队中显示为证据精读。覆盖率的分母必须使用唯一 DOI 数；团队贡献判断则保留多条关系记录。

### 5.7 第二批逐图审计新增的条件边界

第二批 5 篇样本故意覆盖 FSP1–CoQ 防御、氧化 PE 分子种、ER 空间模型、外源 MUFA 状态重塑和溶酶体铁化学。它们彼此并非都能被压成同一条线性机制，因此新增以下规则：

1. **信号、增敏物和充分执行因子必须分开**：LiperFluo/C11 信号升高只是读出；某氧化 PE 与死亡共变是候选信号；预形成脂质只在 RSL3 背景下增强死亡时，应写成“增敏/放大”，不能写成“单独足以执行死亡”；
2. **空间结论必须附条件向量**：每条“起始于 ER/溶酶体/质膜”的结论都要同时记录诱导剂、细胞类型、探针、时间窗和是否使用强制定向分子。不同论文的空间结论冲突时，不投票选一个器官，而是比较条件；
3. **外源营养脂质是暴露实验，不是膳食建议**：必须记录标称浓度、白蛋白/脂蛋白载体、预处理时间、细胞系和进入膜脂所需酶。需要 6–10 小时和 ACSL3 的 MUFA 保护属于脂质状态重塑，不是急性自由基清除；
4. **强制定位证明局部可行性，不等于内源起点**：细胞器定向内过氧化物或 Fento-1 能证明某区室足以承载相关化学，但不能替代无扰动条件下的自然时序与必要性；
5. **局部给药不是系统药理**：淋巴内、瘤内或器官局部给药可以建立局部 proof-of-concept，但治疗窗、全身毒性和药代仍为 Unknown；
6. **模型图不是新的因果证据**：论文只有三张数据主图时，可把关键 Extended Data 纳入证据单元；综合模型图必须标成作者解释，不能为了满足固定 Figure 数量而冒充实验。

这批论文还暴露出一个重要的团队层结论：Kagan/Stockwell 的 ER 相关模型、Dixon 的质膜终末损伤和 Yamada/Rodriguez 的溶酶体模型目前更像是“诱因 × 时间 × 探针 × 细胞状态”的条件函数，而不是一个已经统一的器官顺序。FerroScope 应追踪哪些新实验能够真正区分这些模型，而不是提前制造共识。

### 5.8 第三批逐图审计新增的压缩失真检查

第三批 5 篇样本故意沿着“膜接近性 → 过氧化生成 → 自由基测量 → 链反应阻断 → 抗氧化再生”选择，而不是继续堆叠热门团队。它们分别是 GPX4 fin-loop、POR、NBD-Pen、Fer-1/Lip-1 和 FSP1–维生素 K，新增以下规则：

1. **反应型探针是化学参与者，不是摄像机**：NBD-Pen 以捕获脂质来源自由基解除淬灭，因此会消耗被测物。必须记录探针浓度、分配、反应竞争和加合物；探针信号不能直接等同于无扰动自由基通量，更不能把一篇未研究 ferroptosis 的方法论文倒推成 ferroptosis 体内验证；
2. **遗传必要性不等于直接酶化学**：POR knockout 降低氧化 PE 并抵抗多类诱导剂，支持其位于过氧化上游，但没有鉴定 CYP/其他电子受体、直接底物或体内通量。筛选命中、上游定位和直接催化必须作为三个不同结论层级；
3. **动力学排序必须带相环境**：Fer-1/Lip-1 在非极性均相溶剂中只是中等链断裂抗氧化剂，在磷脂双层中的表观效率明显不同。所有 `k_inh`、化学计量数和结构—活性结论都必须附溶剂、脂质组成与膜分配条件；
4. **疾病 signature 不是疾病病因**：GPX4 R152H 模型与 Alzheimer’s disease 通路或蛋白组重叠，只能支持“部分分子终点相似”。若没有 amyloid/tau 病理、人群证据和时序干预，不能写成散发性 AD 由 GPX4 定位缺陷驱动；
5. **超营养剂量和预防性处理不是饮食或治疗建议**：维生素 K 动物保护使用超营养剂量或损伤前处理，高浓度时还存在 FSP1 非依赖还原途径；必须与普通饮食暴露、损伤后治疗、凝血效应和华法林相互作用分开；
6. **模板槽位必须与 canonical metadata 一致**：期刊、年份、文章状态和 Figure 数不能继承上一条内容。`Cell` 论文出现“Why Nature/Why Science”必须报错；Figure 模型图不得被压缩成新增实验；
7. **更正要按影响层级处理**：方法/试剂目录更正、图例更正与改变结论的更正不是同一风险。POR 论文 2021 年更正只涉及抗体目录信息，因此记录版本但不降级主结论；若更正改变关键数据或撤回，则必须同步改变审计状态。

这批审计进一步说明：60 秒读法的价值是高效生成“值得验证的问题”，不是生成可以直接发布的事实；单篇逐图读法负责找出压缩时丢失的条件；团队纵向读法负责判断这些条件是一次性技术限制，还是持续影响该实验室结论边界的结构性问题。

### 5.9 第四批逐图审计新增的疾病与转化边界

第四批 5 篇样本覆盖原位肺癌、淋巴结转移生态位、移植器官保存、肿瘤免疫和肾损伤。它们显示“模型更接近临床”并不会自动缩短因果链，反而需要额外区分以下层级：

1. **肿瘤启动期遗传删除不是已建立肿瘤治疗**：与肿瘤诱导同步发生的 knockout 可证明肿瘤建立依赖，却会混合起始、选择和维持效应；必须用已建立肿瘤中的药理、诱导删除和救援另行验证治疗主张；
2. **体外阴性不等于体内无依赖**：FSP1 等依赖可能只在特定器官、氧张力、营养和免疫环境出现。记录结果时必须附“细胞来源 × 解剖部位 × 培养/体内状态 × 诱导方式”的条件向量，不能用常规培养皿筛选否定体内机制；
3. **免疫系统必须记录细胞类型方向性**：促进肿瘤细胞 ferroptosis 与保护 T 细胞、髓系细胞或正常组织可能方向相反。全身抑制剂/救援剂同时作用多类细胞时，只能支持系统级贡献，不能直接归因到肿瘤细胞；
4. **临床表达或 signature 不是患者机制测量**：队列中的 SLC7A11、SLC3A2、FSP1、MDA 或基因集关联只建立相关性。没有直接氧化脂质、铁依赖、死亡时序和干预，不得写成预测标志物已验证或患者疗效由 ferroptosis 驱动；
5. **离体人器官不是临床试验**：弃用人肺、离体灌流和分割器官设计可提供高价值人源 proof-of-concept，但没有受者移植、排斥、感染和长期功能结局时，不能写成临床有效；
6. **配对器官改善对照，不自动解决小样本**：同一供体双肺分配不同条件能降低供体间差异，但 `n=5` 仍主要是描述性证据，不能用配对设计掩盖统计不确定性；
7. **多机制化合物要拆分作用归因**：同一化合物可能同时表现为链断裂抗氧化剂、铁结合/再分布剂和抗炎分子。除非有靶点占领、结构—功能分离和遗传互作，不得指定单一分子靶点或把全部器官保护命名为 ferroptosis 抑制；
8. **连续体内选择会混合适应与克隆选择**：多轮转移选择后再离体扩增可得到稳定表型，但不能单凭终点组学重建患者转移的自然时序；应加入谱系追踪、单细胞克隆和可逆性实验；
9. **Corrected proof 与 Publisher Correction 必须持续追踪**：文章上线、正式卷期、更正稿和后续编辑声明属于版本链。更正不改变结论时仍要记录具体改动；涉及结构、样本标签或关键图版时，必须重新核对逐图档案，而不能只保留 DOI；
10. **局部给药成功不是全身治疗窗**：瘤内、淋巴内或器官灌流能证明局部可行性；解剖部位差异、NSG/免疫完整模型差异、药代与正常组织毒性仍需分别回答。

这批样本也补足了团队纵向读法：单篇论文告诉我们某个模型中“发生了什么”，而跨论文与官网对照才能判断团队是否正在稳定积累同一问题所需的模型、化学和临床资源。团队 `currentSignal` 必须描述相对旧主线的新增能力或边界，不能只改写最新论文标题。

### 5.10 第五批逐图审计新增的机制充分性与争议链规则

第五批 5 篇样本覆盖性激素调控的膜脂重塑、DHODH–CoQ 防线、细胞接触传播、线粒体条件性和 ALDH2*2 心肌缺血。它们要求三维读法同步工作：60 秒层先识别最值得验证的主张，逐图层判断必要性/充分性和剂量条件，团队纵向层再判断它是稳定主线、独立建组前工作还是存在后续争议。新增规则如下：

1. **完整 Figure 访问是逐图标签的硬门**：只有摘要、新闻稿、数据库片段或章节摘录时，最多标为 `evidence-audited`。MARCH7 论文虽与铁稳态高度相关，但本轮未获得可核对的完整主图，因此不为凑数升级；宁可保留缺口，也不根据摘要虚构 Figure chain；
2. **增益筛选和过表达不等于内源充分性**：CRISPRa 命中和强制表达能发现潜在防线，却不能证明所有组织的生理表达量足以承担该功能。必须补充内源 loss-of-function、回补、底物依赖和组织表达条件；
3. **总磷脂重塑不等于氧化脂质通量**：PE–MUFA 增加、PE–PUFA 减少可支持底物池变化，但若 LC–MS 主要测未氧化总分子种，就不能写成已经直接测得致死氧化 PE 下降；
4. **已有临床药物加实验性 FIN 不等于可立即再利用**：enzalutamide 或 fulvestrant 已获批，不会自动让其与 IKE、RSL3 或工程化 GPX4 KO 的组合变成临床可行方案。两端药物状态、体内暴露、免疫背景和正常组织治疗窗必须分别记录；
5. **争议药理必须记录剂量—脱靶—遗传证据三角**：DHODH 研究中 brequinar 等高浓度条件可能同时抑制 FSP1。2023 年 Matters Arising、作者 Reply 与原论文必须组成同一版本/争议链；作者更正若只改资助声明，不得与机制争议混写。最终结论应写成“遗传证据支持条件性贡献，药理归因在高浓度下受 FSP1 脱靶挑战”；
6. **细胞器必要性必须以诱导方式索引**：Parkin–CCCP、TCA/ETC 抑制证明线粒体参与 cystine deprivation/erastin，不代表 RSL3 或 GPX4 KO 同样依赖线粒体。任何“某细胞器是 ferroptosis 核心”都必须附 FIN 类别、营养状态、时间和细胞背景；
7. **工程化触发与最小膜重构证明的是层级化充分性**：Opto-GPX4Deg 证明可选择触发，人工 bilayer/GUV 证明接触膜传播具有 physicochemical sufficiency；二者都不能单独证明该机制在真实组织中占主导，也不能排除其他条件下的 ROS 波或局部可溶信号；
8. **人类基因型关联与小鼠机制是三角互证，不是临床闭环**：ALDH2*2 人群血浆氧化脂质关联，加上 knock-in 小鼠、靶向 oxPL 和生化互作，可显著提高可信度；但观察性人群、术前 Fer-1 和预防性 AAV9 干预仍不能给出患者发病后的治疗窗口；
9. **广谱翻译因子和代谢蛋白必须保留多效性负担**：eIF3E 调节多种 mRNA，ALDH2 同时承担醛脱氢作用。即使 TFRC/ACSL4 形成漂亮因果链，也必须问分离功能突变、全局翻译副作用和 4-HNE 等并行损伤；
10. **团队纵向归属不能被高影响论文覆盖**：Gao 的 2019 线粒体论文是独立建组前第一作者工作，Jiang 为通讯作者；它可证明 Gao 的个人机制积累，但不能冒充 HIT 实验室独立产出。团队档案应把“个人研究谱系”和“独立团队当前能力”分开。

本批还确立一个覆盖率原则：逐图精读数量只统计已读完整图文的唯一 DOI，不因同一论文被多个实验室引用而重复，也不因论文重要就降低访问门槛。覆盖率应让缺口可见，而不是制造看似完整的一手资讯幻觉。

## 6. 数据状态

团队档案使用两个阶段：

- `screened`：官网可达，已形成问题和监控问题，但重大论文尚未逐篇核验；
- `audited`：官网研究方向与至少两篇原创研究已交叉核验，并记录限制和审计分数。

`screened` 不等于“已完成精读”。前端必须把两者明确区分。

## 7. 审计评分（100 分）

- 元数据与版本状态：15
- 团队归属与官网交叉验证：15
- 科学问题清晰度：15
- Claim–evidence 对应：25
- 限制、反证与条件边界：20
- 对本项目的可操作价值：10

低于 70 分不得显示“已精读”；70–84 为“有条件通过”；85 以上为“通过”。

## 8. 迭代顺序

1. 先用 5 个能力互补团队建立首批审计样本；
2. 运行结构与内容测试，记录模板失败点；
3. 将 37 个团队全部纳入 `screened`；
4. 按相关度、更新速度和证据风险逐批升级为 `audited`；
5. 每批至少加入一条反证/限制检索；
6. 微信范例正文或人工批注更新后，升级方法版本并重新跑测试。

当前 v0.8 已获得三份用户提供的阅读范例正文，但只抽取思维框架，不导入无关学科内容。五批共 25 篇唯一论文已完成逐 Figure 原文核对，并据此加入阅读深度、工具扰动、工程模型、疾病外推、混合死亡程序、唯一论文身份、空间条件向量、营养暴露、局部给药、直接酶化学、膜相动力学、免疫细胞方向性、离体器官证据层级、争议药理、诱导方式索引和更正影响分级等边界；后续仍应把高相关代表论文逐批升级，不能把二次精读文本视为金标准事实库。
